# Decision Log

Captures terminology, structural, and positioning decisions with rationale. Intended to survive across sessions so future contributors understand *why* choices were made.

| # | Date | Decision | Rationale | Alternatives Considered |
|---|------|----------|-----------|------------------------|
| 1 | 2026-03-19 | Replace "reliable" with "durable" in the five system properties (safe, secure, durable, auditable, observable) | "Reliable" is generic — every framework claims it. "Durable" adds a distinct architectural property: systems that survive failures, maintain state across interruptions, and preserve governance guarantees over time. The term resonates with the practitioner audience who understands durable execution. | Keep "reliable" (defensible but weak); drop to four properties (tighter but loses a real concern); "accountable" (human property, not a system property — mismatch in a list describing system characteristics) |
| 2 | 2026-03-19 | Replace "prior art" with "existing work" throughout core docs | "Prior art" is patent/legal jargon that creates unnecessary friction for non-academic readers. "Existing work" is immediately clear to any audience — security practitioners, compliance teams, executives, engineers. | "Foundations" (good but implies hierarchy); "related work" (academic convention, same jargon problem); "building blocks" (too informal for framework tone) |
| 3 | 2026-04-21 | **AGF Positioning Pillars.** Six pillars define what AGF is: (1) Category stake = Agentic Governance (narrower than AI governance); (2) Four verbs — Synthesizes / Unifies / Prescribes / Operationalizes (causal order); (3) OTAA invariant — every AGF primitive MUST be Observable, Traceable, Auditable, Agent-operable; (4) Dual-form principle — "machine-consumable, human-decidable" (every primitive in human and machine form simultaneously); (5) Tempo taxonomy — wire-speed enforcement (<1ms), near-realtime supervision (sec–min), human-speed governance (hours–weeks); (6) NIST CSF parallel — AGF Rings are to agentic governance what CSF Functions are to cybersecurity. | Supersedes earlier "integration layer" framing. "Integration layer" remains accurate as a description of what AGF *does for a reader*, but "synthesis + unification + prescription + operationalization" is what AGF *is*. Different audiences, not in conflict. | "AI governance framework" (too broad, blends with every competitor); "framework of frameworks" (accurate-but-weak, no commitments); "responsible AI framework" (wrong category — RAI is upstream of agentic). |
| 4 | 2026-04-21 | **Seven-layer stack: where control catalogs and risk quantification fit.** AGF sits at Layer 0 as architectural substrate. Layer 1 = OWASP Agentic Top 10 (threat baseline). Layer 2 = CSA MAESTRO + Microsoft Failure Mode Taxonomy (threat modeling). Layer 3 = control catalogs — CSA AICM, ISO 42001/27001, NIST 800-53, EU AI Act Annexes, BSI AIC4. Layer 4 = CSA Agentic Trust Framework (operating model). Layer 5 = Microsoft AGT + CAF + Agent 365 (runtime reference). Layer 6 (orthogonal) = risk quantification — FAIR, FAIR-CAM, ISO 31000, NIST AI RMF Measure. AGF primitives crosswalk to Layer 3 catalogs; AGF does not replicate catalog content. FAIR and related methodologies are methodology, not primitive — they integrate via AGF's Decision Intelligence capability. | The landscape is crowded. Without an explicit stack, any organization asks "do I need AGF or do I need X?" With the stack, the question resolves: you need AGF *and* X, because they live at different layers. Layer 0 names AGF's position unambiguously. | Absorb FAIR as a primitive (conflates methodology with pattern); position as a control catalog (AGF is not a catalog — see entry 6); flat "framework ecosystem" diagram (no architectural story, no composition logic). |
| 5 | 2026-04-21 | **Dual-form is load-bearing at gate boundaries.** Refinement of the dual-form principle. Every gate decision output (PASS / REVISE / HALT / GATE / ERROR) MUST emit both a human-readable rationale AND a machine-readable artifact simultaneously — same decision, two audiences, same moment. | Dual-form is a general principle for all primitives; at gate boundaries it becomes non-negotiable. A gate that emits only human-form forces machine systems to do NLP; a gate that emits only machine-form leaves auditors with raw JSON. Both forms required, always, at gates. Affects Primitive #8 (Governance Gates), the Composability Interface spec, and Primitive #11 (Trust Ladders, whose promotions/demotions are gate decisions). | "Machine form at gates only when needed" (weakens the invariant — once optional, rapidly skipped); "human form as a derived artifact" (breaks OTAA agent-operability if machine form isn't the primary emission); no gate-boundary invariant at all (leaves dual-form as aspirational). |
| 6 | 2026-04-21 | **Maturity Model — rename L1 to "Non-existent"; absorb inventory/classification content into L2 Foundation.** AGF retains its 5-level program maturity model. L1 is renamed from "Awareness" to "Non-existent" to honestly name the state most organizations are in today. The old L1 content (agent inventory exists, basic risk classification) is absorbed as L2 Foundation entry criteria. L2–L5 labels unchanged. AGF's L1–L5 is a program-level scale complementary to — not substituted for — CSA ATF's 4-tier per-deployment autonomy scale (Intern → Junior → Senior → Principal). | "Awareness" implies prior investment (inventory, classification) that most organizations with agents in production have not made. "Non-existent" is the honest floor. Makes L1 → L2 Foundation the headline governance-investment story and gives AGF a clear ask. | Keep "Awareness" (dishonest about the starting state); adopt full CMMI 0–5 scale (discards AGF's specific L2–L5 semantics that the community has started adopting); add a new L0 (breaks the 5-level structure). |
| 7 | 2026-04-21 | **Harness = governance envelope around an AI agent (source + runtime adapters, inseparable).** Formalizes "harness" as an AGF term with the industry-standard meaning. A harness is the declarative source artifacts (intent, constraints, capabilities, policy, evaluation criteria) PLUS the SDK-specific runtime enforcement adapters that translate them into deterministic controls at call time. Formula (industry convention): `Agent = Model + Harness`. Tiered model per AGF Agent Harness Spec v0.3 — Tier 0 (Provider, precondition) / Tier 1 (Agent, maps to AGF Security Fabric) / Tier 2 (Orchestration, maps to AGF Security Governance). Distinct from Wrapper (one deployment mode), Hypervisor (AGT-specific runtime isolation kernel), and Test Harness (general software engineering). | Industry has converged on this definition across Anthropic ("Agent = Model + Harness"), Parallel Web Systems, Philschmid, Atlan, and the Kim & Hwang SSRN paper. AGF's own Agent Harness Spec v0.3 uses the same framing. Earlier internal notes that drew a hard line between design-time scaffolding and runtime enforcement were inconsistent with industry usage and the spec; the Source → Build → Runtime pattern lives *within* the harness, not as a distinction between harness and something else. | Reserve "harness" for design-time scaffolding only (rejected — conflicts with industry usage and AGF's own spec); avoid naming "harness" at all (rejected — it's becoming a load-bearing term in the field; AGF needs a canonical definition); treat "harness" and "wrapper" as synonyms (rejected — wrapper is one deployment pattern; harness is the governance envelope regardless of deployment pattern). |
