AGF is maintained by a single author in public, with explicit confidence levels and acknowledged open questions. The roadmap below reflects the current state of work after v0.2.0; it is updated when meaningful items ship.

This page is a public summary of where AGF is heading. Major decisions surface in [Decisions](/docs/transparency/decisions). Released work surfaces in the [Changelog](/docs/transparency/changelog).

## Now (in flight)

<Confidence level="informed">

- **First adopter case study.** AGF currently has zero published implementations. Coordinating a written walkthrough of an actual deployment (artifacts, caveats, outcomes) is the highest-leverage remaining work for moving AGF from "serious synthesis" to "publicly cited reference." This is the structural blocker for full Public Readiness closure.
- **Agentic Compliance Blind Spots analysis.** A public gap analysis surfacing where current control catalogs (CSA AICM, NIST 800-53, ISO 42001) leave agent-specific failure modes uncovered. Operationalizes the "gap disclosure" commitment in [Relationship to Frameworks](/docs/reference/relationship-to-frameworks).

</Confidence>

## Next (queued, not yet started)

<Confidence level="informed">

- **GDR machine-form artifacts.** Reference [Governance Decision Record](/docs/reference/governance-decision-record) JSON Schema validators, example GDR libraries for the canonical scenarios, and FAIR-CAM control-effectiveness templates that consume GDRs. Closes the gap between AGF's "machine-consumable" architectural commitment and the artifacts that make it operational.
- **Maturity-level conformance criteria.** Concrete, testable criteria for advancing through the AGF program-maturity levels (L1 Non-existent → L5 Optimized). Today the levels are labeled and characterized; they are not yet independently auditable.

</Confidence>

</Confidence>

## Not on the roadmap (intentional)

<Confidence level="established">

- **AGF is not a runtime.** No reference implementation is planned. Install Microsoft AGT, AWS-native agent governance, or whatever runtime your organization converges on.
- **AGF is not a certification.** CSA STAR for AI and ISO 42001 are the relevant assurance programs.
- **AGF is not a control catalog.** AICM, NIST 800-53, and ISO 42001 are the catalogs.
- **AGF is not consensus.** AGF is synthesized by a single author; the BDFL governance model is intentional. See [`GOVERNANCE.md`](https://github.com/jessepike/agf/blob/main/GOVERNANCE.md).

</Confidence>

## How priorities are set

Items move from **Soon** → **Next** → **Now** when they pass three gates:

1. **Coherence** — does the change reconcile to existing canonical primitives, or does it require a new Decision entry?
2. **Confidence** — can the change ship with a defensible confidence label (`established` / `informed` / `open`)?
3. **Concreteness** — is the deliverable a specific artifact (doc, schema, diagram, primitive) rather than a research direction?

If you want to influence the roadmap, the highest-leverage paths are: contribute evidence from a real implementation, challenge a `<Confidence level="informed" />` claim with counter-evidence, or open a discussion at [GitHub Discussions](https://github.com/jessepike/agf/discussions).
